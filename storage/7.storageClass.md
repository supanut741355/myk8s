# Without storageClass

(Static provisioning volume)
- In the case we create PVC from gcloud: persistent disk.
- Before the PV is created, we need to 1st provision the persistent disk on Gcloud.
- And then create pv-definition file  by using the same name disk 
```
 $ gcloud beta compute disks create \
  --size 1GB \ 
  --region asia-southeast1\
  pd-disk
```



Q: How to PV get volume automatically when app require? 
A: storageClass

# StorageClass
(Dynamic provisioning volume)
- Define provisioner eg. gcloud persistent disk.
- Attach to the pod when claim is made.

When we apply the storageClass
- We no longer need the PV definition (automatically created).
