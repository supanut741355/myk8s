# Persistent Volume Claims

PVC is responsibility to make storage available to pod.


1. Admin create PV.
2. User create PVC to use stroage.

when PVC is create the k8s will binding the PV and PVC

*Every PVC is bound to single PV

# To binding the PVC and PV
- use label and Selector


pv
```
lables:
  name: pv-vol01
```


pvc
```
selector:
  matchLabels:
    name: pv-vol01
```

- PV and PVC is 1-1 relationshi
- If there no volume or PV available -> PVC will remain pending util PV and PVC are available


# Delete PVC

persistentVolumeReclaimPolicy: [TYPE]
- Retain
- Delete


(Retain)
- If we delet PVC previous PVC binding PV -> It will back to stage **Retain** => Keep PV and data

(Delet)
- Delete PV
