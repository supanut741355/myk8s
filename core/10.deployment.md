# Deployment

using for real workload for production deployment.

Deployment can handle feature more than pod and replicaset:
- Deploy versioning 
- Scaling 
- Resource limit
- Deployment strategy (rolling upgrade, blue-green)

Deployment control replicaset that control the pod.


When we apply the deployment resource we will get:
- deployment
- replicaset
- pod


the format of deployment manifest same as replicaSet but diff kind
```
apiVersion: apps/v1
kind: Deployment
metadata:


spec:
  template
  ""
    pod
  ""

  replicas:
  selector:
    matchLabels:
  


```


### Require field
- **spec.selector** is required
- **spec.template.metadata.labels** is require
