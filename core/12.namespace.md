# Namespace (ns)

3 ns k8s created automatically
- default
- kubesystem
- kubepublic


Usecases:
- one cluster has multi env with namespace (ns: dev, ns: stg, ns: prod)



Features:
- Assign quotas resources of namespace. **(Namespace - resource limit)**


To communicate **same** namespace with DNS
```
<name>:<port>
```

To communicate **across** namespace with DNS
```
<name>.<namespace>.svc.cluster.local:<port>
```


k8s cli flg
```
-n <ns_name>


$ kubectl get pod -n <namespace>


$ kubectl apply -f <yyyyy>.yml -n <namespace>
```

Can define the namespace at manifest file at metadata section
```
apiVersion: v1
kind: Pod
metadata:
  name: myapp-pod
  namespace: yoyo
  labels:

spec:
```

flag all namespace
```
--all-namespace or -A

eg.

$ kubectl get pod -A
```


Get current context and namesapce
```
$ kubectl config current-context
```


Switch to specific namespace
```
$ kubectl config set-context $(kubectl config current-context) -n yoyo
```
