# Pod

pod = encapsulated of app's container, will be place on worker node

The simplest of run application is "The cluster we have 1 node and has 1 pod (container in side) is running"


# In the case of scale
Increase the pod, if resource is not enough will increase node 1st and create pod after that.

# Multi-container pod
Actually the normally case signle pod will have one container. With the rare case the pod can has 2+ container this call "Sidecar" 
eg.

pod has 2 container 
- container 1 is app A
- container 1 is app B

app A and B act and communicate like a local has
- share network
- share storage
- same namespace
- 2 container will create together and destroy together.
