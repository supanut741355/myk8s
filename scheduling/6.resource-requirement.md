# Resource Requirement

Many nodes in cluster has a set of CPU, memory resource available

the k8s resource is created, required a set of resource to run.
eg.

pod require 2 mi and 100Mi




the Kube-scheduler will take consideration the amount of resource require by a pod availbe on the node and identifine the best node to place it.

so if node have no sufficient resource available, the kube-secheduler avoide placing and instead place the node where sufficient resource.

and in the case every node is full (not have enough resource), the k8s-resource will in **Pending state** -> if we check with $ kubectl describe, we will get the cause of issue.

eg.
- insufficient CPU


---

# Request

We can specify the resource of each k8s-resource by key:
```
resources:
  request:
    memory: "4Gi"
    cpu: 2
```

cpu minimum is 1 minicore (1mi)

memory:
- 1 Ki
- 1 Mi 
- 1 Gi


# Limit

by default is no limit resource = it can consume all the node both of CPU and memory 
```
resources:
  limits:
    memory: "8Gi"
    cpu: 4
```


---

# The Exceed limit
Q: when pod use exceed resource beyond the specify limit ?
A: 
- in case of **CPU** -> the system will **throttle cpu** = container cannot use more CPU resource than limit

- in case of **Memory** -> container will use more memory resource than limit but the pod will be **terminated** aka **Out of memory kill(OOM)**

---

# Behavior of CPU

### case 1: with out request and limit
result: 1st pod can consume all the cpu resource on node, and prevent the 2nd pod of require resource. => this not work on production


### case 2: only limit, free request (request = limits)
result: k8s will automatically **set request same as limit**.


### case3: Set request and limit
result: Each pod will guranteed the number of CPU, memory request and limit
= can use the resource to ceiling


### case4: Set request but not limit (this most ideal setup)
result: the limit not set = 1st pod can consume all cpu as available, but any point of time if 2nd pod require additional cpu it will be guranteed request CPU cycle
