How scheduling works


k8s has add key **nodeName** to manifestfile

eg.
pod-manifest.yml
```
apiVersion: v1
kind: Pod
metadata:

spec
  containers:
  - name:
    image:
  nodeName: node02
```


k8s will read this nodeName key to plan to **bind pod to node**
1. check node has existing
2. deploy pod to node

---

if no scheduler -> the pod will continue **pending** state

so can manually assign pod to node urself:
1st: Ez way is set **nodeName** file to manifest file


2nd: Create Binding obj and send POST req to pods binding api
with specific target.name

```
apiVersion: v1
kind: Binding
metadata:
  name: nginx
target:
  apiVersion: v1
  kind: Node
  name: node02
```

can execute curl
```
$ curl --header "Content-Type:application/json" \
  --request POST \
  --data '{"apiVersion":"v1", "kind": "Binding" ...}' \
  http://$SERVER/api/v1/namespaces/default/pods/$PODNAME/binding
```
