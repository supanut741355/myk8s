# DaemonSet

like a replicaSet as help to deploy pod run **one copy of pod on each node** in cluster.

whenever a new node is added to cluster, pod is automatically added to that node. And when node is removed, the pod is automatically removed. (= not worry about adding or removing the node)


DeamonSet ensure that one copy of pod alway present in all node in cluster.


perfect usecase:
- Monitoring solution -> deploy monitoring agent
- Logs viwer -> log collector
- Network solution -> calico
on each of node in cluster.

use-case of k8's resource itself:
- kube-proxy



# Cli
create daemonSet
```
$ kubectl apply -f xxxx-damonSet.yml
```

view daemonSet
```
$ kubectl get deamonsets


$ kubect describe daemonset <metadata.name>


```
