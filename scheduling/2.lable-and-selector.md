# Labels and Selector in K8S
we create a lot of resource type in k8s (pod, svc, rs, deployment) that need the way to filter and view diff object categories

eg.

- group object by type (pod, svc, deployment)
- group object by application (appA, appB)
- group object by functionality (frontend, backend, db)


# Filter

We will add labels at manifestfile's **metadata** section

app=App1

```
apiVersion: v1
kind: Pod
metadata:
  name: simple-app
  labels:
    app: App1
    function: front-end

spec:
  conatiner:
  - name: simpale-app
    image: simple-app
```


To filter can use
```
$ kubectl get pods --selector app=App1
```

# Connect obj together
eg. bind replicaSet with pod 

```
appVersion: apps/v1
kind: ReplicaSet
metadata:
  name: simple-app-rs
  lables:
    app: App1
    function: front-end

spec:
  replicas: 3
  seletor:
    matchLabels:
      app: App1
    template:
      metadata:
        labels:
          app: App1
          function: front-end
      spec:
        containers:
        - name: simple-app
          image: simple-app
```

eg2. bind svc to pod
```
apiVersion: v1
kind: Servie
metadata:
  name: my-service
  labels:
    name: simple-app-svc
    function: front-end

spec:
  selector:
    matchLabes:
      app: App1
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080

```

# Annotation

- while label and selector are used to group and select obj.
- annotation used to record other detail for **info** purpose.

info such as: tool detail name, version, build info

eg.
```
appVersion: apps/v1
kind: ReplicaSet
metadata:
  name: simple-app-rs
  lables:
    app: App1
    function: front-end
  annotations:
    buildversion: 1.34

spec:
  replicas: 3
  seletor:
    matchLabels:
      app: App1
    template:
      metadata:
        labels:
          app: App1
          function: front-end
      spec:
        containers:
        - name: simple-app
          image: simple-app
```
